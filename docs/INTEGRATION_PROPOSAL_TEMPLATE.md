# Hapa Integration Suggestion Template

Use this template to suggest a bounded exploration between Hapa and a for-profit team, nonprofit organization, public-interest project, research group, creator collective, or independently operated service.

This is a conversation starter, not a contract, partnership announcement, endorsement, acceptance, compatibility claim, funding commitment, or promise of commercial participation. Omit confidential information, credentials, personal data, private customer information, and binding terms from public issues or discussions.

Attribution inheritance means preserving these source and lineage facts through supported downstream records. It does not transfer ownership or create endorsement.

## Suggested first response

Hapa can initially classify a suggestion as:

- `needs_context` — useful direction, but required boundaries are missing;
- `candidate` — suitable for a bounded design or discovery pass;
- `prototype_ready` — smallest proof, ownership, authority, and verification are clear;
- `deferred` — useful but not currently resourced or aligned;
- `declined` — outside scope, unsafe, incompatible, or not mutually useful.

None of these states creates a partnership or commercial commitment. Only a separately documented agreement and verified implementation can establish those claims.
